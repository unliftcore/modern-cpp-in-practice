附录：术语对照表与翻译约定

本附录用于统一简体中文译稿中的术语、语气与表达方式。所有章节都应遵守这里的约定，以保证整套译文在技术含义、措辞风格和读者体验上保持一致。

翻译约定

  • 面向有经验的程序员，语气直接、克制、技术化，不添加鼓励式口吻或教程式铺垫。
  • 优先保留设计压力、约束、权衡、失败模式和工程后果，不把原文改写成松散的概念介绍。
  • 代码、类型名、库名、标准术语、文件名、链接路径和 Markdown 结构保持原样；仅翻译读者可见的自然语言内容。
  • 对于在中文技术语境中已有稳定译法的术语，采用常见且精确的译名；对容易歧义的术语，首次出现时优先使用本表定义的固定译法。
  • 需要强调风险、边界或审查判断时，使用简洁、确定的表述，不弱化原文判断力度。
  • 追求“信、达、雅”，但以“信”和“达”为先：先确保语义准确，再追求中文行文自然。

核心术语对照

英文统一译法说明
ownership所有权指对资源有效性与释放责任的承担。
borrowing借用指不接管生命周期的使用。
lifetime生命周期统一译为“生命周期”，避免与“作用域”混用。
contract契约指类型或函数在边界上的承诺。
invariant不变量保持数学与工程语义一致。
value semantics值语义标准译法。
identity身份当对象“作为其自身”重要时使用。
failure boundary失败边界不译为“故障边界”，以贴近软件错误处理语境。
boundary translation边界转换指在边界处转换失败表示。
undefined behavior未定义行为缩写仍可写作 UB。
reviewable可评审指足够清晰，便于审查者推理。
structured concurrency结构化并发采用并发领域常用译法。
cancellation取消指主动停止无效工作的请求。
suspension boundary挂起边界保留与协程语义的一致性。
contention争用用于锁、核心时间、缓存行等竞争。
backpressure背压采用业界常用译法。
throughput吞吐量标准译法。
locality局部性指硬件访问局部性。
cost model成本模型指时间、内存、同步等具体成本来源。
hot path热路径标准译法。
vocabulary type词汇类型指用于表达边界契约的标准类型。
observability可观测性标准译法。
diagnostic build诊断构建指以发现和解释问题为目标的构建配置。
type erasure类型擦除标准译法。
call site调用点根据上下文可写作“调用处”，默认使用“调用点”。
shutdown关闭指系统或组件进入停止过程。
tradeoff权衡统一不用“折中”。
failure mode失败模式指设计在什么情况下出错。
ownership transfer所有权转移与 move 语义配套使用。
shared state共享状态并发章节统一译法。
tooling工具链 / 工具结合上下文判断，涉及构建与诊断体系时优先“工具链”。
